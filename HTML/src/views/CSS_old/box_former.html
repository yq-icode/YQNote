盒模型

对象的实际高度
margin-top + border-top + padding-top + height + padding-bottom + border-bottom + margin-bottom
边距碰撞
当元素的下边距碰到了另一个元素的上边距时，浏览器不是把这两个值相加，而是会使用最大的那个边距；
浏览器显示差别
FF: 设置 padding和border 后, div 会增加 height 和 width, 但 IE 不会, 故需要用 !important 多设一个 height 和 width, 即在IE中width包括paddig与border , 因此增加与减少padding 与 border都不影响DIV的宽度；
p标签亦有此属性;
按钮是唯一 一个不遵守盒模型的元素

表单元素

IE6 IE7 IE8 FF
padding border margin padding border margin padding border margin padding border margin
inputbox 1 2 auto 1 2 auto 1 2 auto 1,0 2 0
textarea 1 2 auto 2 2 auto 2 1 auto 0 2 1,0
select 0 2 auto 0 2 auto 0 1 auto 0 1 0
总结：
inputbox,textarea遵守盒模型，即：占用的实际范围 = padding + border + width(css中设置的宽度值)
IE6,IF7,IE8,FF都遵从此效果。
select不遵守盒模型，即：占用的实际范围（=css中设置的宽度） = border + padding + width(实际宽度)
E6,IF7,IE8,FF都遵从此效果。

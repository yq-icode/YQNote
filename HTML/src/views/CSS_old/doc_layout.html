CSS文档统筹

1. 合理地分离样式表文件Top

根据页面类型分离文件
例如根据网站的首页、频道页面和详细页面设计不同的样式表文件,当网站的页面类型比较单一,显示样式又比较统一时,适合选用这种方案.
根据功能模块分离文件
根据页眉、页脚、导航条、功能块等设计不同的CSS文件. 缺点：会产生很多个很小的CSS文件,因为每个页面可能包含很多模块,导致每个页面都包含一堆CSS文件,给管理带来一定麻烦.
根据标签类型分离文件
根据HTML标签的不同类型来分离CSS文件,如把与FORM表单相关的样式代码放在一个文件中,把与A相关的链接样式放在另一个文件中.如果一个网站中有50个页面,其中12个页面含有FORM,就适合使用这种方案. 缺点：会使样式表文件变得很细碎,一个页面有时会导入很多个样式表文件.
根据设备类型分离文件
根据打印、手提设备和屏幕等多种媒体类型设计不同的样式文件.
@import
@import可以在一个CSS文件或HTML文档中包含很多其它样式表文件,因此设计师可以细致分离样式文件,再通过组合并用@import把需要的文件导入到一个新的CSS文件中,这样就只需把这个新的包含文件导入到不同的网页中就可以实现统一管理,用这种方法可以创建网站的主CSS文件或实现CSS文件的多种组合,实现代码的充分利用.当网站中每个页面都导入很多个不同的CSS文件时,应该考虑使用这种方法.
根据代码规模分离文件
这种方法是对前面几种方法的综合,设计师可以根据代码规模和意图,综合利用上面介绍的方法,实现CSS代码的有机分离.

2. 规则的组织Top

即先定义哪些样式,后定义哪些样式,选择符之间如何分离等. 先定义基本元素属性,然后设计布局属性.在布局时先定义总体结构,然后再分块详细定义.

3. 属性的组织Top

按主次关系组织：先声明布局属性,然后定义盒模型属性,最后再定义排版属性.

布局属性：
display list-style position float clear
盒模型属性
width height margin padding border background
排版属性
color font-size text-align text-decoration white-space等

4. CSS命名方法Top

语义化命名：使用可以描述元素所在位置的语义词汇来对其命名
id=”left-side” id=”center-column”
使用颜色名称命名颜色:
.blue{color:blue;} .f00{color:#f00;} .abc123{color:#abc123;}
使用”font+字体大小”命名字体大小:
.font12px{ font-size:12px;} font9pt{font-size:9pt;}
使用英文名称命名:
.left{float:left;}
结构化命名：根据文档结构来命名
id=”main_nav” id=”sub_nav” id=”main_content”

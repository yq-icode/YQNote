attr与prop的区别

区别一

prop
prop是从属性对象中取值，属性对象中有多少属性，就能获取多少值，不需要在页面中显示定义。
所以，即使#chk1 元素上并没有书写 checked 属性，但是 $("#chk1").prop("checked") 仍获取到了，这是因为 checked 是 checkbox 的固有属性；
attr
attr 是从页面搜索获得元素值，所以页面必须明确定义元素才能获取值，相对来说比较慢。
所以，$("#chk1").attr("checked") == undefined，无法获取到 checked 属性。
总结
如果是自定义的属性，那么属性对象中是没有这个属性的。所以prop返回undefined。但是页面中可以检索到这个属性，所以attr是可以获取的。
	                
<input id="chk1" type="checkbox" data-state="0" />
<input id="chk2" type="checkbox" checked />
<input id="chk3" type="checkbox" checked="checked" />
		            
	            
	                
//固有属性             	
$("#chk1").prop("checked") == false
$("#chk2").prop("checked") == true
$("#chk3").prop("checked") == true

$("#chk1").attr("checked") == undefined
$("#chk2").attr("checked") == "checked"
$("#chk3").attr("checked") == "checked"

//自定义属性
$("#chk1").prop("data-state") == undefined
$("#chk1").attr("data-state") == 0

区别二

prop
prop属性值是动态的，比如checkbox，选中后，checked变为true，prop值也会发生改变。
attr
attr获取的是初始化值，除非通过attr('name','value')改变，否则值不变。

即当通过点击切换 checkbox 的选中状态时，在点击事件中通过 prop("checked") 获取 checked 属性时，属性值会随着点击而改变，即选中时为 true, 未选中时为 false; 但是在点击事件中通过 attr("checked") 获取 checked 属性时，checked 属性值是一直不变的。

最终结论

  • 对于HTML元素本身就带有的固有属性，在处理时，使用prop方法。
  • 对于HTML元素我们自己自定义的DOM属性，在处理时，使用attr方法。
